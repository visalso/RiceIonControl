Detailed Description

This class contains a list of WaveformGenerator objects and a Timing object.

It is intended to represent a chasis with a NiSync Card and a number of PXI-6733s. It contains low level functions for the Wavefrom Chassis class.

Chassis.git.WaveformChassis.WCLowLevel.loops

The number of times to iterate over a Finite number of samples.

This value is only useful in the "Finite" mode. It is the number of times that a sequence of voltages will be looped. The default is allways 1.

Chassis.git.WaveformChassis.WCLowLevel.mode

This is the mode of operation for all generators.

There are currently three modes available. Static mode is where one static voltage is set with no need for a sample clock. Finite mode is where a finite number of voltages will be set at a sample clock rate. Continuous mode is where a sequence of voltages are generated at a sample rate and then repeated until the stop() method is called.

Chassis.git.WaveformChassis.WCLowLevel.startTriggerSource

This is the start trigger source terminal for every WaveformGenerator configured in the WaveformChassis.

It is defaulted to "PXI_Trig1"

Chassis.git.WaveformChassis.WCLowLevel.timing

This is a Timing object created using the Timing class.

It is used to synchronize all of the PXI cards.

Chassis.git.WaveformChassis.WCLowLevel.triggerType

The trigger type for the analog outputs.

There are currently two trigger types - "Software" and "Hardware." The "Software" mode means that analog output channels are not syncronized. While "Hardware" means that analog output channels are syncronized to a start trigger. The startTriggerSouce attribute must be configured appropriately.

Chassis.git.WaveformChassis.WCLowLevel.useTiming

This is a boolean value that will be true if the WaveformChassis is configured to use Timing.

This property contains the data that was last stored on the buffer. It does not query the device for what is on the buffer, so you must write to the buffer for the data to be returned accurately.
